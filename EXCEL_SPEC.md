# 엑셀 업로드 스펙

대시보드가 기대하는 엑셀(.xlsx) 파일의 시트와 컬럼 구조를 정의합니다.

> 모든 처리는 **브라우저 안에서** 일어나며 서버로 전송되지 않습니다.

## 시트 구성

3개 시트가 필요합니다 (`Customers`는 선택, 매니저 강점 분석의 고객 프로필 비교에서만 사용).

### 1. `Transactions` 시트 (필수)

거래 1건 = 1행.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `거래ID` | 문자열 | ✅ | 고유 키. LineItems / Customers와 JOIN |
| `날짜` | 날짜 | ✅ | 거래 일자 (`2024-01-15`, `2024/1/15`, Excel serial 모두 허용) |
| `지점` | 문자열 | ✅ | 지점명 (예: `강남`, `송파`) |
| `지도사` | 문자열 | 선택 | 매니저(장례지도사) 이름 |
| `패키지` | 문자열 | 선택 | 패키지명 (예: `기본화장`, `올인원`, `자연닮음`, `요람장`, `관장례`, `소동물_기본`, `소동물_관`). 비어있으면 추모전용으로 간주 |
| `장례매출` | 숫자 | ✅ | 장례 카테고리 매출 (원) |
| `부가매출` | 숫자 | 선택 | 부가서비스 매출 (원) |
| `무게추가` | 숫자 | 선택 | 무게 추가 매출 (원) |
| `할인금액` | 숫자 | 선택 | 적용된 할인액 (원) |
| `추모전용` | bool/숫자 | 선택 | `1`/`true`/`예`면 장례거래 분모에서 제외 (봉안당만 / 산골만 등) |

### 2. `LineItems` 시트 (필수)

한 거래에 포함된 라인아이템(개별 상품/옵션) 1건 = 1행.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `거래ID` | 문자열 | ✅ | Transactions와 매칭 |
| `카테고리` | 문자열 | 선택 | `업셀링` / `스톤` / `봉안` / `기타` 등 (보조 분류용) |
| `품목` | 문자열 | ✅ | 품목명. 아래 분류 참고 |
| `가격` | 숫자 | ✅ | 실제 결제가 (원) |
| `기본가` | 숫자 | 선택 | 해당 품목의 패키지 기본가. 가격 > 기본가면 "업셀링 발생" |
| `수량` | 숫자 | 선택 | 기본 1 |

#### 품목 분류

대시보드의 매트릭스/표는 아래 카테고리를 인식합니다.

| 그룹 | 인식 품목명 |
|---|---|
| 업셀링 5품목 (가격 차액 기반) | `관`, `요람장`, `수의`, `수의보`, `유골함` |
| 스톤 | `스톤제작`, `스톤함` |
| 봉안 | `봉안당신규`, `봉안당연장` |
| 기타 부가 | 카테고리=`기타` 또는 `기타부가` 로 표시된 모든 품목 |

> 품목명은 위 문자열과 **정확히** 일치해야 합니다. 다른 명칭을 쓰는 경우 엑셀에서 미리 치환하거나 `lib/schema.ts`의 상수를 조정하세요.

### 3. `Customers` 시트 (선택)

매니저 강점 분석 탭의 "Top vs Bot 고객 프로필 비교"에서만 사용됩니다. 없으면 해당 카드만 비활성화됩니다.

| 컬럼명 | 타입 | 필수 | 설명 |
|---|---|---|---|
| `거래ID` | 문자열 | ✅ | Transactions와 매칭 |
| `보호자연령` | 숫자 | 선택 | 만 나이 |
| `반려동물체중` | 숫자 | 선택 | kg |
| `반려동물품종` | 문자열 | 선택 | 품종명 |
| `사망원인` | 문자열 | 선택 | 자유 텍스트 (TOP 12 집계) |

## 자주 묻는 사항

- **컬럼 순서**: 무관. 헤더(첫 행)의 이름으로 매칭합니다.
- **빈 값**: 숫자는 0으로 처리됩니다.
- **시트명 변형**: `Transactions` 대신 `transactions`/`거래`/`거래내역` 도 인식.
- **파일 크기 한계**: 브라우저 sessionStorage 한계(보통 5~10MB JSON)를 넘으면 변환 후 오류가 납니다. 대형 데이터는 기간을 나눠 업로드하거나 JSON 사전집계 방식 사용.

## 사전 집계된 JSON 업로드

이미 같은 형태로 가공된 `DATA` JSON이 있다면 1단계를 건너뛰고 2단계의 "JSON 파일 선택"으로 바로 로드할 수 있습니다. JSON 구조는 `lib/schema.ts`의 `DashboardData` 타입 참고.
